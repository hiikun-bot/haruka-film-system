メンバー登録の手順

招待リンクの発行から、初回登録、プロフィール初期設定まで。
はじめてシステムを使う人もここを見れば一通り終わります。

01 / OVERVIEW

① 登録は2人で進める作業

メンバー登録は 「招待する側（管理者・秘書）」 と 「招待される側（新メンバー本人）」 の2人で進めます。
本ガイドはどちらの立場の人にも対応しています。該当する側だけ読めば OK です。

📨 招待する側
管理者・秘書。メンバー画面から招待リンクを発行し、Chatwork や LINE などで本人に送ります。
→ Part A へ
📩 招待される側
新しく入る本人。受け取ったリンクから自分で表示名・パスワードを決めて登録します。
→ Part B へ
💡 所要時間の目安： 招待発行 約2分 ／ 本人登録 約3分 ／ 初期設定 約5分。合計 10 分程度で完了します。
02 / ROLES

② ロール（権限）の選び方

招待時に ロール を1つ選びます。これは「その人がシステム上で何をできるか」を決める重要な設定。
あとから管理者が変更できますが、まず近いものを選びましょう。

✂️ 編集者 editor
動画編集を担当。クリエイティブの制作・納品・自分の請求書作成。動画編集者の標準。
🎨 デザイナー designer
バナー・サムネ等のデザインを担当。基本権限は編集者と同じ。
🎯 ディレクター director
クリエイティブのDチェック・修正依頼。チームの代表 D に指定可能。
📊 プロデューサー producer
案件管理・予算・Pチェック。クライアントとのやり取り。
🔄 P兼D producer_director
プロデューサーとディレクターの両方を兼ねる人向け。両方の権限を持つ。
📋 秘書 secretary
管理者とほぼ同等の権限。メンバー招待・マスター編集が可能。
👑 管理者 admin
全機能・全データへのフルアクセス。基本は経営層のみに付与。
⚠️ ロール選びの基本： 迷ったら 「編集者」 を選んでください。あとから昇格は簡単ですが、いきなり強い権限を渡すと事故のもとです。
03 / PART A 📨 招待する側（管理者・秘書）

③ 招待リンクを発行する

メンバー画面から招待リンクを発行し、Chatwork / LINE / メール などで本人に送ります。

1
メンバー画面を開く
PC：上部ナビ「メンバー」
スマホ：ハンバーガーメニュー →「👤 メンバー」
💡 表示されない？ 招待権限がないアカウントです。管理者・秘書ロールでログインしてください。
2
「＋ 招待リンク発行」をクリック
画面上部のツールバーにある ＋ 招待リンク発行 ボタンを押すと、発行モーダルが開きます。
画面イメージ
3
メールアドレスとロールを入力
招待先のメールアドレスと、付与するロールを選択します。
入力イメージ
招待先メールアドレス *
権限
⚠️ 注意： メールアドレスは 本人が普段使う実在のもの を入力。間違えても招待は飛ばないので発行はできますが、ログイン時の本人識別子になるので慎重に。
4
「リンクを発行」を押す
招待メッセージが自動生成されます（24時間有効の招待 URL 入り）。
生成されるメッセージの例
【HARUKA FILM SYSTEM へのご招待】

この度、チームメンバーとしてご参加いただくことになりました。
以下のリンクより登録手続きをお願いいたします。

▼ 参加登録URL（24時間以内に登録をお願いします）
https://haruka-film-system.up.railway.app/invite.html?token=...

登録完了後、Googleアカウントまたはパスワードでログインできます。
権限：編集者

ご不明な点はお気軽にご連絡ください。
よろしくお願いいたします。
5
「📋 メッセージをコピー」 → 本人に送信
コピーしたメッセージを Chatwork ／ LINE ／ メール ／ Slack DM など、普段使っている連絡手段で本人に送ってください。リンクには 24 時間の有効期限があります。
🔒 セキュリティ： 招待リンクは 第三者に絶対に転送しないこと。リンクを知る人なら誰でもそのメールアドレスでアカウントを作成できてしまいます。
04 / PART B 📩 招待される側（新メンバー本人）

④ 招待リンクから登録する

受け取った招待リンクを開き、自分で表示名とパスワードを決めて登録します。

1
招待リンクを開く
Chatwork や LINE で受け取った https://...invite.html?token=... のリンクをタップ／クリックしてください。
⏰ 24時間ルール： 発行から 24時間 を過ぎたリンクは無効になります。期限切れの場合は招待者に再発行を依頼してください。
2
登録画面で内容を確認
画面上部に「📩 招待が届いています」と あなたのメールアドレス ＋ 付与されるロール が表示されます。間違いがないか確認してください。
❌ 間違っていたら登録しないで！ メールアドレスやロールが違う場合は、その場で招待者に連絡を。誤った状態で登録してしまうと、管理者にロール変更を依頼する必要があります。
3
表示名・パスワードを入力
3項目を入力します。
項目
説明
表示名
他のメンバーから見える名前。本名 or 普段呼ばれている名前（例：田中 太郎）。あとで変更可。
必須
パスワード
8文字以上。英数字混在を推奨。あとで変更可。
必須
パスワード（確認）
同じパスワードをもう一度入力。打ち間違い防止。
必須
4
「アカウントを作成してログイン」を押す
ボタンを押すと自動的にログインされ、ダッシュボード画面に遷移します。ここまでで登録は完了です 🎉
💡 ログインは2方法： 次回以降は ① メール＋パスワード、② Google アカウント連携 （同じメールアドレス）のどちらでも入れます。

06 / PART D ⚙️ 仕上げ（管理者・秘書）

⑥ 管理者がやる仕上げ作業

本人が登録を完了したら、管理者が ランク・所属チーム・必要なら単価上書きを設定して完了です。

1
メンバー一覧で本人を編集
メンバー画面 → 該当メンバーの行の「編集」ボタン → 基本情報タブ。
2
ランク（S / A / B / C）を設定
編集者・デザイナーは ランクで単価が決まります。クリエイティブ単価マスター（マスター画面）と組み合わせて自動計算。
💡 ランクの基準： 経験年数・実績・社内評価で決まる。判断が難しい場合はまず B から始めて様子を見るのが無難。
3
所属チーム（基本）を設定
どの基本チームに所属するか選びます。代表 D / 代表 P が自動的にデフォルト承認者になります。
⚠️ 1人1チーム： 兼務は不可。複数チームに関わる場合は、案件ごとに「案件専用 D / P」を上書き指定してください。
4
（任意）クリエイティブ単価の上書き
特定の案件・特定のクリエイティブで、ランクとは違う単価を使いたい場合は、案件画面の「クリエイティブ単価」から個別に上書き可能。変更時は理由必須（監査ログに残る）。
07 / FAQ

⑦ トラブルシューティング

招待リンクを開いたら「期限切れ」と出ました。
招待リンクは発行から 24時間で無効 になります。招待者に 再発行 を依頼してください。同じメールアドレスで再発行可能です。
「すでに使用されています」と出ました。
そのリンクはすでに登録に使われています。すでにアカウントが作成済みなので、ログイン画面 からメール＋パスワードで入ってください。パスワードを忘れた場合は管理者に パスワードリセット を依頼。
「このメールアドレスはすでに登録済み」と出て招待発行できません。
過去に同じメールで登録された人がいます。メンバー一覧 で該当メンバーを検索 → 既存アカウントを使うか、不要なら無効化してから別メールで招待してください。
登録したけど通知（DM）が来ません。
本人プロフィールの 連絡先タブ で Slack DM ID または Chatwork DM ID が空欄の可能性大。プロフィール編集 → 連絡先タブで入力してください。それでも来ない場合は、Bot がそのワークスペース／ルームのメンバーになっているか管理者に確認を。
ロールを間違えて発行・登録しました。
登録自体はそのままで OK。管理者に依頼すれば、メンバー編集画面でロールを変更できます。アカウントを作り直す必要はありません。
退職メンバーのアカウントはどうすれば？
削除ではなく 無効化（is_active = false） がおすすめ。過去の請求書・コメント等の参照が壊れません。メンバー編集モーダル右上のメニューから無効化できます。
08 / DONE

⑧ 完了チェックリスト

これが全部終わったら、メンバー登録は 完璧 です 🎉

📨 招待する側

  • 正しいメールアドレスとロールで招待リンクを発行した
  • 招待メッセージを本人に送信した（24時間以内に登録するよう伝えた）
  • 本人の登録完了を確認し、ランク・所属チームを設定した
  • 必要なら案件レベルでの単価上書きを設定した

📩 招待される側

  • 表示名・パスワードを設定し、登録を完了した
  • プロフィール ＞ 基本情報：アバター・ニックネーム・職種を入力
  • プロフィール ＞ 稼働時間：平日／土日祝の稼働時間帯を入力
  • プロフィール ＞ 連絡先：Slack ID と Chatwork ID を入力（DM 通知のため）
  • プロフィール ＞ 口座情報：振込先口座と請求書用住所を入力（編集者・デザイナーは必須）
  • データ構造ガイド を一読し、システムの全体像を把握した
