1💬 Slack DM ID
なぜ入れる？

システムから あなた宛て に Slack DM を飛ばすための宛先 ID。

入れるとどう変わる？

Dチェック依頼／修正通知／納品確認が あなたの Slack に直接届く。「気づかずスルー」がなくなる。

📋 例：あなたのクリエイティブが「Dチェック」に進んだ瞬間、ディレクターに Slack DM 通知 → 確認スピードアップ
取得方法

Slack の DM 画面 → プロフィール右クリック → 「メンバーIDをコピー」。U から始まる 10 桁前後の文字列。

2💬 Chatwork DM ID
なぜ入れる？

Chatwork メンションの あなたの番号。Chatwork を主に使うクライアント・チームで重要。

入れるとどう変わる？

Chatwork の自分宛てに通知が飛ぶ。Slack 派／Chatwork 派どちらでも漏れなく届く。

📋 例：Chatwork メンション形式 [To:6603336] の 数字部分 を入力
取得方法

Chatwork のメンションをコピペ、または プロフィール画面の「アカウント ID」を確認。

1📷 アバター（プロフィール画像）
なぜ入れる？

クリエイティブ一覧やコメント欄で、誰の作業／発言かが パッと見で分かる ようにするため。

入れるとどう変わる？

アサイン候補リストで 名前より先に顔が記憶されるので、声がかかりやすくなる。

📋 例：D が編集者を選ぶとき、アイコンが入っている人から目に止まる → アサイン率アップ
2🏷 ニックネーム
なぜ入れる？

本名と普段呼ばれている名前が違うとき、検索・メンションで 呼び名でも見つかる ようにするため。

入れるとどう変わる？

「田中太郎」の登録で「たろ」を入れておけば、メンバー検索で「たろ」と打っても出る。連絡漏れを防ぐ。

📋 例：「たろさんに連絡したい」→ メンバー検索 たろ でヒット
3💼 職種（動画／デザイン／両方）
なぜ入れる？

システムが 「この人にどのアサインを薦めるか」 を判定するため。

入れるとどう変わる？

動画案件のアサイン候補に名前が出る／デザイン案件は出ない、と 自動で振り分けられる。
「両方」を選べば両方の案件で候補に上がります。

📋 例：D が動画編集者をアサイン → 「動画」or「両方」を選んだ人だけがリストに出る
4🎂 誕生日
なぜ入れる？

社内の誕生日通知に使われます。年は 非表示にする設定 も用意（月日のみ表示）。

入れるとどう変わる？

誕生日当日にチームから祝ってもらえる。会ったことがないリモートメンバーとも距離が縮まる。

📋 例：誕生日当日朝に Slack で「今日は田中さんの誕生日 🎉」と通知
あとでもOKな項目：すべて（必須は登録時の名前・パスワードのみ）

1🕘 平日 稼働時間
なぜ入れる？

D／P が「夜遅い時間にアサインしていいか」を判断するため。

入れるとどう変わる？

稼働時間外の急な依頼が減る。あなたが寝ている時間帯に「明日朝までにお願い」が来なくなる。

📋 例：22 時に修正依頼を出すか迷った D が、稼働「09:00 〜 21:00」だと分かれば → 翌朝に回す判断ができる
2🌅 土日祝 稼働時間
なぜ入れる？

土日に動ける人かどうかをチームが知るため。「稼働なし」も明示できる（チェック 1 つ）。

入れるとどう変わる？

金曜遅くに飛び込んできた緊急案件で、土日稼働可能なメンバーから優先的にアサイン候補 に出る。

📋 例：金曜 18 時に CM 公開直前修正が発生 → 土日「稼働あり」のメンバーで素早くチーム編成
3📝 メモ（スキル等）
なぜ入れる？

得意なソフト・ジャンルをチームに伝える自由記述欄。プロフィール一覧で表示される。

入れるとどう変わる？

ニッチなスキルが必要な案件で 指名されやすくなる。連絡可能時間帯を補足するのにも使える。

📋 例：「After Effects 得意」と書いておくと、AE 必須の案件で D が「この人に頼もう」と即決
4👁 クリエイティブ初期ビュー
なぜ設定する？

クリエイティブ画面を開いたときの 初期表示フィルタ。3 つから選べる。

入れるとどう変わる？

「自分のボール」 を選ぶと、今あなたが動かす番のクリエイティブだけ が最初に並ぶ。仕事の優先度が一発で分かる。

📋 例：朝出社して画面を開いた瞬間、「今日やる作業」が 3 件だけ表示 → 余計なものが視界に入らず集中できる
✅

STEP 3

口座情報を登録する

⏱ 約 3 分

納品 → 自動で請求書ドラフト → 月末に自分で「提出」 → 振込 という流れ。
振込先として口座情報が必須です（編集者・デザイナー）。

← 左の ① 〜 ④ をタップしてみてください なぜ入れる？／何がよくなる？／例
1🏦 銀行・支店・口座番号
なぜ入れる？

請求書承認後の 振込先。これがないと支払いが止まる（編集者・デザイナーは必須）。

入れるとどう変わる？

月末に請求書を提出 → 管理者承認 → 自動的にこの口座に振込まれる。手作業のやり取りなし。

📋 例：5 月末に請求書提出 → 6 月 10 日にこの口座へ着金
2✏️ 口座名義（カナ）
なぜ厳密？

銀行の振込照合は 名義カナの完全一致 をチェック。1 文字でも違うとエラーで戻ってくる。

入れるとどう変わる？

正しく入れておけば一発で振込完了。間違えると振込が戻ってきて支払い遅延 になる。

📋 NG 例：通帳「タカハシ　サトル」(全角スペース) を「タカハシ サトル」(半角) で入力 → 振込エラー
👉 通帳・キャッシュカードを見ながら正確に！
3📞 電話番号
なぜ入れる？

請求書フォーマットの 請求者連絡先 として印字される。

入れるとどう変わる？

クライアントから見える正式な書類になる。経理問い合わせ時の連絡先として機能する。

📋 例：請求書の右上「請求者」欄に表示
4🏠 住所
なぜ入れる？

請求書の 正式な発行者住所。税務上必要な情報として保存される。

入れるとどう変わる？

請求書フッターに住所が印字される。確定申告時の控えとしても残る。

📋 例：請求書フッター「〒123-4567 東京都渋谷区...」と表示
✅
口座情報、登録完了！これで請求書がスムーズに処理されます
