SYSTEM GUIDE

データの紐付け（リレーションシップ）図解

HARUKA FILM SYSTEM では「クライアント」「案件」「チーム」「メンバー」「訴求軸」「商材」がツリー構造で繋がっています。
このページは新メンバーへの説明資料としてお使いください。

01 / HIERARCHY

① 大きな入れ子構造

まず一番大事なのは 「クライアント > 案件 > チーム > メンバー」 の入れ子。
下の階層は必ず上の階層に 1つだけ 紐づきます（メンバーは1チーム所属、案件は1クライアント所属）。

🏢 クライアント CLIENT
お金を払ってくれる発注元の会社。「誰の制作物を作っているか」のいちばん外側の枠。
例：◯◯化粧品株式会社
1社につき N 件
📁 案件 PROJECT
納品の単位。1つの目的・キャンペーンで作る制作物の束。締切・予算・担当 D/P もここで決まる。
例：2026 夏キャンペーン動画
担当チームを N 個アサイン
👥 チーム TEAM
D/P/編集/デザインがセットになった制作チーム。代表 D・代表 P を持ち、案件の通知先のデフォルトになる。
例：青チーム（代表D：田中、代表P：佐藤）
1チームに N 人
👤 メンバー MEMBER
あなた。1つのチームに所属し、ロール（編集者/ディレクターなど）と単価ランクを持つ。
例：山田太郎（編集者・Aランク・青チーム所属）
02 / RELATIONSHIP

② 紐付けの3つの基本パターン

「リレーションシップ」と呼ぶときは、この3パターンのどれかを言っています。
システム上のすべての繋がりは、この3つの組み合わせで出来ています。

PATTERN A

1 対 1 (1:1)

1つのAに対して、1つのBだけが紐づく関係。

例：メンバー ↔ Slack ID（1人につき1つ）
PATTERN B

1 対 多 (1:N)

1つの親に、複数の子がぶら下がる関係。このシステムの中心構造。

例：1クライアント → N案件 ／ 1案件 → Nクリエイティブ
PATTERN C

多 対 多 (N:N)

双方向に複数紐づけられる関係。

例：1クライアントを N チームが担当 ／ 1チームが N クライアントを持つ
03 / PROJECT-CENTERED

③ 案件のまわりに集まるすべてのもの

案件は 「ハブ」。クライアント・チーム・クリエイティブ・訴求軸・商材すべてが案件に集まります。
「迷ったらまず案件画面を見ろ」と覚えてください。

クライアント：案件はどのクライアントの仕事かを必ず1つ持つ。
チーム：誰が制作するか。チームの代表 D/P が案件のデフォルト担当。
クリエイティブ：案件の中で作る動画・デザイン1本ごと。バージョン履歴付き。
訴求軸：何を訴求するか（例：保湿力・低価格 など）。クライアント or 案件で管理。
商材：どの商品を売るか（例：化粧水・乳液 など）。クライアント or 案件で管理。
04 / SYNC

④ 訴求軸・商材は「親から自動同期」 or 「案件で個別管理」

混乱しやすいポイント。訴求軸と商材はクライアントレベルにも、案件レベルにも持てます。
案件画面の「商材・訴求軸」タブで、どっちを使うかをスイッチで切り替えます。

DEFAULT

同期 ON：クライアントの定義をそのまま使う

案件は クライアントが持つ訴求軸／商材マスター をそのまま参照。クライアント側で追加すれば全案件に反映。レギュラー案件で便利。

例：化粧水ブランドA社のレギュラー動画 → クライアントA社で訴求軸を一元管理
CASE

同期 OFF：案件専用で個別に持つ

案件ごとに 独自の訴求軸／商材 をぶら下げる。スポット案件や、特殊なキャンペーンで使う。クライアントマスターは汚さない。

例：A社の単発コラボ案件 → この案件専用の「コラボ商品」を案件側に登録
05 / WALKTHROUGH

⑤ 具体例で追ってみる

「化粧品ブランドA社」のキャンペーン動画案件を例に、紐付けがどう繋がるかを上から下まで追います。

EXAMPLE

A社・夏キャンペーン2026動画

1
🏢 クライアント：A社（化粧品ブランド）
A社のマスターに、商材「化粧水・乳液・美容液」と訴求軸「保湿力・透明感」を登録済み。
2
📁 案件：夏キャンペーン2026動画 ← A社配下
案件の「商材・訴求軸」タブは 同期 ON。A社のマスターをそのまま使う。
3
👥 担当：青チーム ← 案件にアサイン
青チームの代表 D（田中）と代表 P（佐藤）が、案件のデフォルト承認者になる。
4
👤 編集：山田太郎 ← 青チーム所属
山田は青チーム所属の編集者（Aランク）。案件のクリエイティブにアサインされる。
5
🎬 クリエイティブ：化粧水_保湿訴求_15秒
ファイル名は自動生成 ← 📦 化粧水 + 🎯 保湿訴求 + 尺
6
山田が制作 → Dチェック（田中）→ Pチェック（佐藤）→ クライアント確認 → 納品 → 山田の請求書ドラフトに自動追加
06 / MEMBER VIEW

⑥ あなた（メンバー）から見たつながり

逆向きにも辿れます。あなたから案件・クライアントに繋がる経路を覚えておくと、画面の「なぜここに表示？」が分かります。

👤 あなた → 所属 → 👥 1つのチーム
👤 あなた → アサイン → 🎬 クリエイティブ複数 → 📁 案件 → 🏢 クライアント
⚠️ あなたが「画面に見えるクリエイティブ」は、あなたが アサインされている分＋（権限により）チーム全体／全社のもの。
通知（ChatWork DM・Slack DM）は、案件の D/P アサインがあなたに当たっているときに飛びます。
07 / FAQ

⑦ よくある「つまずき」

案件画面に商材／訴求軸が表示されません。
案件タブ「商材・訴求軸」が 同期 ON（デフォルト）の場合、クライアント側のマスター に登録が必要です。クライアント画面を開いて「商材」「訴求軸」タブで追加してください。
クリエイティブのファイル名が空欄になります。
ファイル名は 商材コード × 訴求軸コード × 尺 から自動生成されます。商材か訴求軸が未選択だと生成されません。クリエイティブ詳細で両方選択してください。
通知（ChatWork DM）が飛びません。
5つすべて揃う必要があります：① クライアント or 案件に chatwork_room_id、② 通知先メンバーに chatwork_dm_id、③ Bot がそのルームのメンバー、④ 対象ステータス遷移である、⑤ 環境変数 CHATWORK_API_TOKEN。1つでも欠けると無言でスキップされます。
同じ人が複数チームに所属できますか？
いいえ。メンバーは1チームのみに所属する設計です（兼務なし）。代わりに、案件ごとに「案件専用の D / P」を上書き指定できるので、必要なら案件側で人をアサインしてください。
「ボール保持者」って何ですか？
クリエイティブ進捗の中で 「いま誰のアクション待ちか」 を可視化した概念です。Dチェック中ならディレクター、Pチェック中ならプロデューサー、修正中なら編集者…のように、ステータスから一意に決まります。
