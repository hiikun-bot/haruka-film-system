STEP 2

大容量でも安心（Drive 直送）

⏱ 約 1 分
⚡ 20MB を超えると自動で「直送モード」

20MB 以上のファイルは、サーバーを経由せず ブラウザから Google Drive へ直接（16MB ずつの分割送信で）アップロードされます。回線が不安定でもチャンク単位でリトライされるので、数十 GB の 4K 動画でも OK。切り替えは全自動で、操作は何も変わりません。

アップロード中…（進捗バーと ✕ キャンセルボタンが出ます）
Drive へ直送中… 62%
🗂 保存先は自動で整理される

アップロードしたファイルは Google Drive の クライアント名 → 案件名 → 年月 → 種別 のフォルダに自動で振り分けられ、閲覧用リンクも自動発行されます。自分でフォルダを掘る必要はありません。

STEP 3

ファイル名は自動生成（リネーム不要）

⏱ 約 1 分
🏷 命名規約は「案件のテンプレート」が決める

クリエイティブの名前は、案件ごとに設定された ファイル名テンプレート（連番・案件名・サイズ・商品名などのトークンの組み合わせ）から自動生成されています。アップロード時はそこに _v2.mp4 のように バージョン番号と拡張子が自動で付く だけ。

手元のファイル名: 最終版_fix2_これでいく.mp4 　　　　↓ アップロードすると自動で… Drive 上の名前: 007_あるる_ドラム_上地無_v2.mp4

💡 手元のファイル名は何でも OK。提出用の正式名はシステムが管理します。Drive 上のファイルを手動でリネームする必要はありません（クリエイティブ名が変わったときも Drive 側へ自動で追従します）。

🔢 番号は提出→指摘の「往復」ごとに上がる

バージョン番号は ラウンド番号（提出とそれに対する指摘のワンセット）と同じです。採番はサーバーが自動で行うので、自分で数える必要はありません。

・提出前なら アップロード取り消し → 同じ番号のまま上げ直し ができます（v2 を取り消して再アップしても v2 のまま）。
・過去バージョンのファイルもカードとして残り、いつでも見返せます。うっかり古い方を開こうとすると「過去のバージョンを開こうとしています」と確認が出るので取り違えも防げます。

STEP 5

アップロード後の自動処理

⏱ 約 1 分
🎞 「再生用の高速版」が自動生成される

動画（MP4 / MOV / M4V）をアップロードすると、バックグラウンドで すぐ再生が始まる高速版（faststart 版） が自動生成されます。

・MP4（H.264 + AAC）→ メタデータの位置を先頭に移動するだけ。画質は完全にそのまま
・MOV / ProRes / HEVC など → 高画質設定で MP4 に変換。解像度・フレームレートは変わりません（720p などへの縮小はしません）

💡 チェック画面のプレビュー再生では高速版が自動で使われます。Drive の原本はそのまま残るので、生成に失敗してもアップロード自体には影響ありません。

✅
アップロード後は待つだけ！高速版の生成状況はファイルカードのバッジで確認できます

▶ 使い方

ファイルのチェック画面（プレビュー）ヘッダーにある 「▶ Premiere」ボタン（動画クリエイティブのみ表示）を押すと、タイムコード付きコメントをマーカーに変換する .jsx スクリプトがダウンロードされます。

Premiere Pro で対象シーケンスを開いた状態で File > Scripts > Run Script File からこのファイルを実行すると、指摘が タイムライン上のマーカー として一括挿入されます。マーカー名がタイムコード、コメント欄に「誰の・どんな指摘か」が入るので、再生しながら 1 件ずつ潰せます。

ここだけは絶対！

マーカーになるのは タイムコード付きのコメントだけ。逆に言うと、チェック側がタイムコードを入れて指摘してくれるほど修正が速くなります。届いた指摘への回答は 編集の進め方ガイド の「↩ 引用」を活用！

📤
